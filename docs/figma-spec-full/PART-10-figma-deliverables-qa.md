# PART-10: Структура файла Figma и приёмка

---

## 1. Рекомендуемая структура страниц (Pages)

1. **00 Tokens** — цвета, текст, радиусы, эффекты.
2. **01 Components** — кнопки, поля, карточки, чипы, навигация.
3. **02 Shell** — шапки, нижний бар, 428px frame.
4. **03 Hub** — все состояния экрана «Герой».
5. **04 Finance** — 4 подвкладки + модалка операции + категории.
6. **05 Body** — главная, питание, тренировки, замеры, настройки.
7. **06 Todo** — три подвкладки.
8. **07 Auth** — лендинг, вход, регистрация.
9. **08 Settings** — заглушка настроек приложения.

---

## 2. Обязательные компоненты (variants)

- `Button`: Primary / Secondary / Ghost; Default / Hover / Disabled / Loading.
- `Card`: Surface / Dashed (квесты) / Tinted Body / Tinted Finance.
- `NavItem`: Active / Inactive (нижний бар + поднавигации).
- `Input`: Text, с ошибкой.
- `ProgressBar`: EXP, kcal, бюджет.
- `Chip` / Pill: стрики, фильтры.
- `Modal` / `BottomSheet`: добавление операции (если используется).

---

## 3. Критерии приёмки дизайна (перед вёрсткой)

- [ ] Все цвета сопоставлены с токенами PART-01 (нет «случайного» #333).
- [ ] Ни одна интерактивная цель меньше 44×44px на мобильных макетах.
- [ ] Нижняя навигация: 5 пунктов, порядок как в PART-02.
- [ ] Модуль «Финансы»: 4 подвкладки нарисованы.
- [ ] Модуль «Тело»: замеры доступны из главной, не как четвёртая нижняя иконка модуля.
- [ ] Экран «Герой» соответствует PART-03 (включая стрики и мини-карточки).
- [ ] RU и EN: проверены переносы на длинных подписях (`tabDevelopment`, длинные суммы).

---

## 4. Передача разработчику

- Ссылка на Figma с **режимом Dev**.
- Экспорт иконок SVG при необходимости.
- Таблица соответствия **Figma style → CSS variable** (один лист).

---

## 5. Регрессии, которых избегать

- Путаница **порядка вкладок** приложения — ломает `goToTab` из героя.
- Отдельный «шестерёнка» в макете без экрана настроек — в приложении шестерёнка ведёт на вкладку 4.
- Два разных источника правды для **пассива** на герою и на финансах — должны совпадать по формуле (PART-08).
