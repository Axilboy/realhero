# Real Hero

Веб-приложение для личного учёта и планирования: **финансы** (счета, расходы/доходы, переводы, инвестиционный портфель с котировками), навигация по разделам **Герой**, **Тело**, **Задачи**, **Действия**. Вход по **email и паролю** (без подтверждения почты).

| Слой | Технологии |
|------|------------|
| Фронтенд | **Vite + React + TypeScript**, React Router |
| API | **`api/`** — Fastify, Prisma, **SQLite** |
| Аутентификация | JWT, сессия в httpOnly cookie `rh_session` |

Дополнительно: **паспорт проекта** — [`ПАСПОРТ_ПРОЕКТА.md`](./ПАСПОРТ_ПРОЕКТА.md), история изменений — [`CHANGELOG.md`](./CHANGELOG.md).

## Возможности (кратко)

- **Герой** — стартовый экран приложения.
- **Финансы** — три подвкладки (свайп / поднавигация):
  - **Главный:** счета, балансы, сводка за месяц, список операций, добавление расхода/дохода/перевода/инвестиции.
  - **Инвестиции:** портфель, оценка стоимости, структура (вклады/карты/акции/облигации/прочее), поиск бумаг (MOEX, крипто), обновление цен при открытии вкладки, купон/дивиденды с подстановкой из MOEX где возможно, доход **с одной бумаги** в год.
  - **Аналитика** — расходы и доходы по категориям за выбранный месяц.
- **Тело**, **Задачи**, **Действия** — заготовки экранов под будущий функционал.

## Локальный запуск (два терминала)

**Терминал 1 — API** (из каталога `api/`):

```bash
cd api
cp .env.example .env
npm install
npx prisma db push
npm run dev
```

**Терминал 2 — фронт** (из **корня** репозитория):

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173` — сначала вход; регистрация: `/register`. В режиме разработки запросы на **`/api`** проксируются на `http://127.0.0.1:3000`.

- Сборка фронта: `npm run build`.
- Подробности по эндпоинтам и продакшену: **[`api/README.md`](./api/README.md)**.

## Два шага: ПК → сервер

**Как устроен репозиторий:** один проект **web-app** — это корень Git (фронт, папка **`api/`**, скрипты **`scripts/`**). В **`api/`** лежит только бэкенд; «заливать только api» не нужно — всё в одном репозитории. PM2 запускает Node из **`web-app/api`**, поэтому после `cd api` легко оказаться там в терминале — но скрипт обновления лежит **на уровень выше**, в **`web-app/scripts/`**.

**1. На своём ПК (Git Bash), из корня репозитория** — сохранить изменения и отправить в Git:

```bash
bash scripts/push.sh
```

С суффиксом в сообщении коммита: `bash scripts/push.sh "финансы"`. Если правок в файлах нет, коммит пропускается, но выполняется **`git push`** (удобно, если коммит уже есть).

То же самое: `bash scripts/quick-push.sh` (старый вызов).

**2. На сервере (Linux, SSH)** — подтянуть код, собрать фронт в **`www`**, обновить API и перезапустить PM2 **одной командой**.

Из **корня** клона (рядом с `package.json` фронта и папкой `api/`):

```bash
cd /PROGS/RH/web-app
bash scripts/server-update.sh
```

Если вы уже в **`api/`** (как часто бывает после `prisma` / PM2), путь `scripts/...` оттуда **не существует** — либо `cd ..`, либо:

```bash
bash update-all.sh
```

(файл **`api/update-all.sh`** вызывает тот же сценарий). Либо из любой папки внутри репозитория: `bash "$(git rev-parse --show-toplevel)/scripts/server-update.sh"`.

Путь к клону замените на свой, если отличается. Статика по умолчанию: **`DEPLOY_WWW=/PROGS/RH/www`** (см. `scripts/deploy-server.sh`). Процесс PM2: **`PM2_NAME=realhero-api`**.

Первичная настройка: nginx **`location /api/`** на порт бэкенда, в **`api/.env`** — **`CORS_ORIGINS`** с вашим доменом и сильный **`JWT_SECRET`**. Подробнее: **`api/README.md`**.
