## 2. Вкладка «Главная» (`FinanceMainPanel`)

### 2.1. Настройки отчётного периода

- Гранулярность: день / неделя / месяц / год / свой диапазон (см. `FinanceReportingGranularity`).
- Для месяца — день «отчётного месяца» (1–28).
- Сохранение через API настроек финансов.

**Макет:** компактный селектор периода + отображение выбранных дат.

### 2.2. Карусель счетов (`FinanceAccountsRow`)

- Карточка счёта: баланс, название, тип; при настройках — **строки доходности** (день/неделя/месяц/год) по флагам карточки.
- Горизонтальная карусель с несколькими счетами.

### 2.3. Плитки отчёта за период

- Доходы, расходы, переводы (и баланс периода) — как в текущей реализации; цвета: доход `--rh-income`, расход `--rh-expense`.

### 2.4. Мотивация «пассив» (`FinanceMotivationStrip`)

- Условие показа: **пассивный поток за месяц > 0** (вклады + купоны/дивиденды из обзора).
- Три слайда: **день / неделя / месяц** — примеры «на что хватает» (контент из логики мотивации, не произвольный текст).

### 2.5. Последние операции

- Список последних движений за период (кратко).

### 2.6. Полная сводка

- На счетах, инвестиции (оценка), **всего**, **пассив ~в месяц**.
- При настроенном бюджете — полоска **«осталось по лимитам»**.
- **Разбивка капитала** (структура).

### 2.7. FAB и добавление операции

- Плавающая кнопка **добавления операции** видна на главной вкладке модуля когда экран финансов активен (`fabVisible`).
- Модальное окно / шторка с **`AddTransactionPrototype`**: тип (доход/расход/перевод), сумма, счёт, категория, дата; вкладки внутри модалки для типов.

### 2.8. Режим удаления счёта и прочие опасные действия

- В коде есть режим удаления с опциями **merge-into** и **purge** — макет должен содержать **понятные подписи**, не только иконки.

## 3. Вкладка «Инвестиции» (`FinanceInvestPanel`)

- Оценка портфеля, структура долей.
- Карусель вкладов/накопительных (аналогично главной по паттерну карточек).
- Метрики из `fetchInvestOverview().metrics`, в т.ч.:
  - на 1000 ₽ в год;
  - %% вкладов в месяц;
  - бумаги: день / месяц / год.
- Блок подсказок по пассиву — **три текстовых блока** (не карусель), если реализовано в коде.

## 4. Вкладка «Аналитика» (`FinanceAnalyticsPanel`)

- Прогноз и разбивка по категориям (как в продукте).
- Макет: достаточно места для графиков/таблиц; конкретный тип графика — ориентир на текущий UI.

## 5. Вкладка «Бюджет» (`FinanceBudgetPanel`)

- Таблица лимитов на **календарный месяц**.
- Строки читаемы; переполнение лимита визуально отличимо.

---

## 6. Дерево категорий (`CategoryTreePanel`)

Согласно `ТЗ_UI_МОКАПЫ_V2.md`:

- Вкладки **Расход / Доход**, поиск.
- Дерево: строки **крупнее**, без «стены» мелких иконок.
- Действия: текстовые **«Подкатегория»**, **«В архив»**.
- Новая категория: поле имени + **«Создать»**; эмодзи и цвет в **раскрывающемся** блоке «Значок и цвет».
- Подкатегория: форма **только название**; цвет/эмодзи от родителя.
