# Техническое задание для Figma AI: шаблон интерфейса Real Hero

**Полная многостраничная спецификация:** каталог `docs/figma-spec-full/` (`README.md` + `PART-01` … `PART-10`) — используйте её как основной источник деталей; этот файл остаётся **коротким брифом** для быстрой вставки в Figma AI.

---

Документ для генерации **дизайн-системы и экранов** веб-приложения **Real Hero** (личный кабинет: финансы, тело, задачи, геймификация). Его можно целиком вставить в Figma AI / аналог как бриф.

**Версия:** 2026-03-29  
**Связанные материалы в репозитории:** `ПАСПОРТ_ПРОЕКТА.md`, `ТЗ_UI_МОКАПЫ_V2.md`, при наличии — `docs/screenshots/v2/*.png` (эталоны).

---

## 1. Цель и результат

**Цель:** получить в Figma **мобильно-ориентированный UI-kit + ключевые экраны** для последующей вёрстки (React), с **единой тёмной темой**, токенами и компонентами (кнопки, карточки, поля, навигация).

**Не цель:** отрисовывать админки, лендинги маркетинга, публичный сайт без авторизации — только **личный кабинет** после входа.

---

## 2. Платформа и ограничения

| Параметр | Значение |
|----------|----------|
| Тип | **Веб-SPA** (браузер), первично **мобильная ширина** |
| Макс. ширина контентной колонки | **428px** (логика iPhone 13 Pro Max), по центру viewport |
| Тема | **Тёмная** (единственная в v2) |
| Безопасные зоны | Учесть **safe-area** (вырез, home indicator) — отступы сверху/снизу |
| Языки интерфейса | **RU** и **EN** (одинаковая вёрстка, разная длина строк) |

---

## 3. Базовые дизайн-токены (v2, согласовать как стили Figma)

Использовать как **стиль/variables**; разработка уже опирается на близкие значения в CSS.

**Фон и поверхности**

- Фон приложения: `#0F1115`
- Карточка / поверхность: `#14171C` … `#1A1D24`
- Граница: `#2A2F38` (при необходимости приглушённая `#252A33`)

**Текст**

- Основной: `#E8EAED`
- Вторичный: `#C4C7CC`
- Приглушённый: `#9AA0A6`

**Акценты**

- Акцент (ссылки, активная вкладка, CTA): `#8AB4FF`
- Доход / позитив: `#81C995`
- Расход / негатив: `#F28B82`
- Акцент «тело» (мягкий): `#B8D4C9`

**Прочее**

- Нижняя панель навигации: фон тёмнее/отдельный токен (например `#12151A`), верхняя граница `#252A33`
- Радиус карточек: **12px** (кнопки ~10px)
- Иконки нижней навигации: монохром **stroke**, активное состояние — акцентный цвет

---

## 4. Типографика (ориентиры)

- Системный стек: **system-ui, sans-serif** (в вёрстке: Segoe UI / Roboto и т.п.)
- Заголовки экранов: жирный, контрастный; подзаголовки и подписи — меньше кегль, secondary color
- Числа в финансах и таблицах: **tabular nums** где уместно
- Не перегружать экран мелким текстом (меньше ~11px) на длинных списках

---

## 5. Оболочка приложения (глобальный layout)

**Всегда присутствует:**

1. **Верхняя панель** (для большинства разделов): слева контекст (email / заголовок раздела), справа **язык RU|EN** и **«Выйти»**. На экране **«Герой»** по референсу v2: слева крупный заголовок **«Герой»**, справа **шестерёнка (настройки)** и **колокол (уведомления, пока заглушка)**; второй ряд — компактно почта + язык + выход (или эквивалент без потери функций).
2. **Основная область** — скролл контента в колонке max 428px.
3. **Нижняя навигация** — **5 пунктов** с иконкой + подпись:

| № | Подпись (RU) | Смысл |
|---|----------------|--------|
| 1 | Герой | Хаб, уровень, виджеты, квесты |
| 2 | Тело | Замеры, питание, тренировки |
| 3 | Развитие | Задачи / рост (rocket-метафора) |
| 4 | Финансы | Счета, операции, инвестиции, бюджет |
| 5 | Настройки | Язык и будущие опции |

Активная вкладка — акцентным цветом.

## 6. Карта экранов (что нужно отрисовать в Figma)

Минимум — **по одному ключевому состоянию** каждого экрана + **пустые/ошибка** где критично.

### 6.1. Герой (Hub)

- Заголовок в шапке как в п.5.
- **Профиль:** аватар-заглушка, строка «Уровень N», **имя пользователя** (из email), титул уровня («Новичок» и т.д.), **полоса EXP** + числа **«текущий / порог EXP»** справа от полосы или в одной строке.
- **Стрики:** две **пилюли** (например: 🔥 тренировки N дней; 📏 замеры N дней), даже при 0 — пилюля, не просто текст.
- **Блок «Сегодня»:** заголовок; справа ссылка **«К задачам»**; два крупных действия: **🏋️ ТЕЛО** и **📝 ЗАДАЧИ**.
- **Две мини-карточки в ряд:**
  - **Тело:** тёплый коричневатый оттенок фона; иконки в шапке карточки; крупная строка ккал **«съедено / цель»** или только факт; **тонкая синяя** полоса прогресса по ккал; строка БЖУ **«Б … · Ж … · У …»** г; три **микро-полоски** (бел/жир/углев) как индикаторы долей от целей.
  - **Финансы:** зеленоватый оттенок; строка **«Всего … ₽»**; при наличии — **«💰 Пассив ~… ₽/мес»**.
- **Квесты:** секция с пунктирной/мягкой рамкой; активный квест: название **«Квест: …»**, **прогресс** справа сверху («Прогресс: x / y»), кнопки **«К задачам»** (первичная) и **«Отказаться»** (вторичная); блок доступных квестов со списком и **«Взять»**.
- Опционально блок **«Скоро»** (инвентарь / питомец / мини-игра) — иконки-заглушки disabled.

### 6.2. Финансы (4 подвкладки)

1. **Главная:** карусель счетов; плитки отчёта за период; при пассивном доходе &gt;0 — мотивационная карусель «на что хватает пассива»; последние операции; сводка (на счетах, инвестиции, всего, пассив ~в месяц); при бюджете — полоска «осталось»; разбивка капитала.
2. **Инвестиции:** оценка портфеля, структура долей, метрики (в т.ч. «на 1000 ₽ в год», деньги с %% вкладов, бумаги день/месяц/год), карусель вкладов/накопительных.
3. **Аналитика:** прогноз, разбивка по категориям.
4. **Бюджет:** таблица лимитов на месяц.

Дополнительно модальные/вложенные: **добавление операции** (тип, сумма, счёт, категория, дата); **дерево категорий** (вкладки расход/доход, поиск, дерево без «стены» мелких кнопок; действия текстом «Подкатегория», «В архив»; создание категории с раскрываемым «Значок и цвет»).

### 6.3. Тело (3 подвкладки + экран замеров)

1. **Главная тела:** вес, график, талия/тренировки за 7 дней, **БЖУ за сегодня**, ссылка на замеры вторичным стилем.
2. **Питание** — день, приёмы, итоги ккал/БЖУ, добавление блюда.
3. **Тренировки** — программы, журнал, старт/завершение тренировки.

**Замеры (отдельный экран):** сверху **дата + вес** обязательны; остальное в сворачиваемом **«Дополнительные поля»** (рост, % жира, талия, заметка).

### 6.4. Развитие (задачи)

- Поднавигация: **Сегодня / Входящие / Все** (или эквивалент на макете).
- Список задач с чекбоксом, сроком, заметкой; пустые состояния.

### 6.5. Настройки

- Язык (дублировать или сослаться на шапку), выход, заглушки под тему/уведомления/экспорт.

### 6.6. Лендинг (до входа)

- Коротко: вход / регистрация (если нужен один экран в наборе «сайта»).

---

## 7. Состояния UI

- **Загрузка:** скелетоны или спиннер в рамках карточки, без сдвига вёрстки.
- **Пусто:** понятный текст + одно основное действие.
- **Ошибка:** красный/вторичный акцент, короткий текст, «повторить» при необходимости.
- **Disabled:** для будущих фич (колокол, «скоро») — пониженная непрозрачность.

---

## 8. Иконки

- Нижнее меню: **Home / Body / Rocket (развитие) / Finance / Settings** — единый стиль **outline**, 22–24px.
- Эмодзи в макете **Герой** допустимы как в референсе (🏋️ 📝 🔥 💰 📊 и т.д.).

---

## 9. Критерии приёмки макета (для дизайнера / AI)

- [ ] Все токены вынесены в **variables** (цвета, радиусы, отступы).
- [ ] Есть **компоненты**: кнопка первичная/вторичная/ghost, поле ввода, карточка, чип, bottom bar, top bar.
- [ ] **428px** frame для основных экранов + вариант **desktop** (колонка по центру на тёмном фоне).
- [ ] Нижняя навигация **5 пунктов** с подписями на RU; длина EN проверена на переносы.
- [ ] Экран **Герой** содержит мини-карточки Тело/Финансы и блок квестов по структуре выше.
- [ ] Экспорт **PNG 2x** или **SVG** для иконок по запросу разработки.

---

## 10. Что передать разработчику из Figma

- Ссылка на файл + **режим Dev** (отступы, шрифты, цвета).
- Экспорт **спецификации токенов** (JSON/таблица) — желательно совпадающая с именами вроде `--rh-bg`, `--rh-accent` для ускорения.
