# Real Hero

Веб-приложение для личного учёта и планирования: **финансы** (счета, расходы/доходы, переводы, инвестиционный портфель с котировками, бюджет по категориям), **тело** (сводка с весом и БЖУ, замеры, дневник питания КБЖУ, программы тренировок и журнал), навигация по разделам **Герой**, **Задачи**, **Действия** (нижнее меню с иконками и подписями). Вход по **email и паролю** (без подтверждения почты). Интерфейс: русский и английский (i18n), целевое описание UI — **`ТЗ_UI_МОКАПЫ_V2.md`**.

| Слой | Технологии |
|------|------------|
| Фронтенд | **Vite + React + TypeScript**, React Router |
| API | **`api/`** — Fastify, Prisma, **SQLite** по умолчанию; **PostgreSQL** — см. `docker-compose.yml` и `api/README.md` |
| Аутентификация | JWT, сессия в httpOnly cookie `rh_session` |

**Версия:** см. semver в `package.json` и `api/package.json` (текущая запись в [**CHANGELOG**](./CHANGELOG.md)). Дополнительно: [**паспорт проекта**](./ПАСПОРТ_ПРОЕКТА.md).

## Возможности (кратко)

- **Герой** — стартовый экран: уровень и EXP, мини-сводки «Тело» (ккал, БЖУ) и «Финансы» (всего, при необходимости пассив ~в месяц), квесты, блок «Скоро».
- **Финансы** — четыре подвкладки (свайп / поднавигация):
  - **Главная:** счета (карусель); плитки доходов/расходов/баланса за отчётный период; при положительном пассивном потоке — **карусель «На что тянет пассивный доход»** (день / неделя / месяц) с примерами покупок на эти суммы; сводка «На счетах», инвестиции, всего, оценка пассивного потока (~в месяц), разбивка капитала, последние операции, бюджетная полоса при лимитах; настройки (отчётность, порядок счетов, доходность на карточках).
  - **Инвестиции:** портфель бумаг, структура долей, вклады с %% (без кредиток и долга по счетам), метрики, поиск котировок (MOEX, крипто), **подсказки по пассиву** (три блока текста, без карусели).
  - **Аналитика** — расходы и доходы по категориям за календарный месяц, прогноз баланса к концу отчётного периода.
  - **Бюджет** — лимиты по категориям на календарный месяц, факт и остаток.
- **Тело** — подвкладки **Главная** (сводка, график веса, БЖУ за день, переход к истории замеров), **Питание** (КБЖУ по дням), **Тренировки** (программы, журнал). Экран **замеров** (вес обязателен, прочие поля в «Дополнительно») открывается с главной тела. Настройки: кг/lb, см/дюймы, цели ккал и БЖУ.
- **Задачи** — «Сегодня» / «Входящие» / «Все», форма новой задачи в карточке; синхронизация с API (`/api/v1/tasks`).
- **Действия** — заготовка экрана под модуль привычек (см. `ТЗ_ДЕЙСТВИЯ.md`).

## API (префиксы)

- **`/api/v1/finance/*`** — счета, операции, категории, сводки, инвестиции, бюджет, настройки отчётности и карточек.
- **`/api/v1/body/*`** — настройки тела, замеры, питание по дню, программы тренировок, журнал (см. `api/README.md`).
- **`/api/v1/tasks`** — задачи пользователя.
- **`/api/v1/hero`** — опыт экрана «Герой» (`GET` и `POST .../sync-from-local` для переноса с `localStorage`); см. `api/README.md`.

## Локальный запуск (два терминала)

**Терминал 1 — API** (из каталога `api/`):

```bash
cd api
cp .env.example .env
npm install
npx prisma db push
npm run dev
```

**Терминал 2 — фронт** (из **корня** репозитория):

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173` — сначала вход; регистрация: `/register`. В режиме разработки запросы на **`/api`** проксируются на `http://127.0.0.1:3000`.

- Сборка фронта: `npm run build`.
- Подробности по эндпоинтам и продакшену: **[`api/README.md`](./api/README.md)**.

## Два шага: ПК → сервер

**Как устроен репозиторий:** один проект **web-app** — это корень Git (фронт, папка **`api/`**, скрипты **`scripts/`**). В **`api/`** лежит только бэкенд; «заливать только api» не нужно — всё в одном репозитории. PM2 запускает Node из **`web-app/api`**, поэтому после `cd api` легко оказаться там в терминале — но скрипт обновления лежит **на уровень выше**, в **`web-app/scripts/`**.

**1. На своём ПК (Git Bash), из корня репозитория** — сохранить изменения и отправить в Git:

```bash
bash scripts/push.sh
```

С суффиксом в сообщении коммита: `bash scripts/push.sh "финансы"`. Если правок в файлах нет, коммит пропускается, но выполняется **`git push`** (удобно, если коммит уже есть).

То же самое: `bash scripts/quick-push.sh` (старый вызов).

**2. На сервере (Linux, SSH)** — подтянуть код, собрать фронт в **`www`**, обновить API и перезапустить PM2 **одной командой**.

Из **корня** клона (рядом с `package.json` фронта и папкой `api/`):

```bash
cd /PROGS/RH/web-app
bash scripts/server-update.sh
```

Если вы уже в **`api/`** (как часто бывает после `prisma` / PM2), путь `scripts/...` оттуда **не существует** — либо `cd ..`, либо:

```bash
bash update-all.sh
```

(файл **`api/update-all.sh`** вызывает тот же сценарий). Либо из любой папки внутри репозитория: `bash "$(git rev-parse --show-toplevel)/scripts/server-update.sh"`.

Путь к клону замените на свой, если отличается. Статика по умолчанию: **`DEPLOY_WWW=/PROGS/RH/www`** (см. `scripts/deploy-server.sh`). Процесс PM2: **`PM2_NAME=realhero-api`**.

Первичная настройка: nginx **`location /api/`** на порт бэкенда, в **`api/.env`** — **`CORS_ORIGINS`** с вашим доменом и сильный **`JWT_SECRET`**. После изменений схемы Prisma на сервере: **`cd api && npx prisma db push`** (или **`npx prisma migrate deploy`**, если ведёте миграции). Подробнее: **`api/README.md`**, опционально **PostgreSQL** — **`docker-compose.yml`** в корне репозитория.
