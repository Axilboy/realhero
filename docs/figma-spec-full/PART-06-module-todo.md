---

## 2. Логика списков (для макета состояний)

- **Входящие:** активные задачи **без** `dueDate`.
- **Сегодня (день D):** просроченные (`dueDate < D`), на сегодня (`dueDate === D`), выполненные в этот день (по `completedAt`).
- **Все:** все незавершённые с сортировкой по сроку и дате создания; завершённые — по политике экрана.

Поля задачи: заголовок, заметка, `dueDate`, `dueTime`, `completedAt`, связь с квестом (`source`, `questInstanceId` — когда есть).

---

## 3. Форма добавления задачи

- Поля: заголовок, заметка, вариант срока (нет / сегодня / своя дата), опционально время.
- Кнопки создания и отмены — по текущему UI.

---

## 4. Интеграция с квестами

- Из экрана «Герой» кнопка **«К задачам»** ведёт на эту вкладку.
- Квестовые задачи визуально можно помечать бейджем (если в коде появится) — см. `ТЗ_КВЕСТЫ_И_ТУДУ.md`.
