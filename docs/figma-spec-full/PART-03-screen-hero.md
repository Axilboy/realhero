# PART-03: Экран «Герой» (Hub)

Источник: `src/screens/HubScreen.tsx`, стили `.hero__*` в `src/index.css`.  
Визуальный эталон: мокап **v2-rh-01** (`docs/screenshots/v2/`, при наличии).

**Актуально для продукта v2:** **`ТЗ_UI_МОКАПЫ_V2.md` §2**, **`ПАСПОРТ_ПРОЕКТА.md` §5.4**, **`ТЗ_ФАЗА_ВИЗУАЛА.md`**.  
Блоки **стриков** и **«Сегодня»** на главной **не используются** (сняты с экрана для снижения перегруза); стрики остаются возможной **будущей** визуализацией в других местах (см. паспорт п. 5.5).

---

## 1. Назначение экрана

- **Точка входа после открытия приложения** (первая вкладка).
- Показывает **прогресс героя** (уровень, EXP), **сводку по телу и финансам** (мини-карточки), **квесты**, блок **«Скоро»**.
- Все числа — **реальные данные** с API (см. PART-08), без фиктивных полей.

---

## 2. Структура сверху вниз (обязательные блоки)

### 2.1. Блок профиля и EXP (`.hero__panel`)

**Не дублировать** заголовок «Герой» в контенте — он в шапке AppShell на этой вкладке.

| Элемент | Описание | Данные |
|---------|----------|--------|
| Аватар | Круг, иконка-заглушка (⚔) | Пока без загрузки фото |
| «Уровень N» | Мелкий серый текст | `splitTotalExp` → `level` |
| Имя | Крупное имя | Локальная часть email до `@` |
| Эпитет | Синий/акцентный титул | По порогам уровня: Новичок, Искатель… |
| Полоса EXP | Горизонтальный прогресс | Заполнение: `expInLevel / expToNext` |
| Числа EXP справа | «`now` / `max` EXP» | Рядом с полосой или в одной строке |
| Подпись | Мелкий текст про синхронизацию | `hub.expFootnote` |

**aria:** полоса — `role="progressbar"`, подпись из `hub.expAria`.

---

### 2.2. Мини-сетка 2×1 (`.hero__minigrid`)

#### Карточка «Тело» (клик → Тело)

- Верхняя строка: эмодзи 🏋️ + заголовок **ТЕЛО** + эмодзи 🔥 по краям.
- **Синяя** горизонтальная полоска прогресса ккал (цель из настроек тела).
- Крупная строка: **`cur / goal` ккал** или только факт ккал, если цели нет.
- Три **микро-трека** для БЖУ (доля от целей белки/жиры/угли — цвета разные).
- Строка БЖУ текстом: «Б … · Ж … · У …» в граммах.

**Фон:** тёплый коричневатый градиент (см. `.hero__mini--body`).

#### Карточка «Финансы» (клик → Финансы)

- Верхняя строка: 💰 + **ФИНАНСЫ** + 📊.
- Строка 1: **«Всего …»** (сумма счетов + инвестиции), без лишнего двоеточия в ключе `financeMiniTotal`.
- Строка 2 (если >0): пассив **«💰 Пассив ~… ₽/мес»** из обзора инвестиций.

**Фон:** зеленоватый оттенок (`.hero__mini--finance`).

---

### 2.3. Квесты (`.hero__card--quest`)

- Заголовок секции: **«Квесты»** (uppercase-стиль в UI).
- Если **нет** активных квестов — показать вводный текст `hub.questText`.
- **Активные квесты:** для каждого:
  - Заголовок **«Квест: {название}»**
  - Справа сверху: **«Прогресс: done / total»**
  - Опционально метка ветки (Обучение, Качалка…)
  - Кнопки: **«К задачам»** (акцент), **«Отказаться»**
- **Доступные квесты:** список карточек с описанием, XP, кнопка **«Взять»**.

---

### 2.4. «Скоро» (`.hero__future`)

- Три иконки-кнопки **disabled**: инвентарь, питомец, мини-игра (эмодзи 🎒 🐾 🎮).
- Только визуальный задел; клик не обязан ничего делать.

---

## 3. Не входит в экран v2 (история / макеты-не-код)

Следующие блоки **описывали** ранние макеты или альтернативные варианты; в **текущем** продукте **не рендерятся**:

- **Стрики** (`.hero__streaks`) — пилюли «тренировки / замеры»; данные по сериям доступны в модулях «Тело» / «Тренировки» и замеры.
- **Блок «Сегодня»** (`.hero__today`) — быстрый переход на Тело/Задачи; переходы доступны через нижнюю навигацию и мини-карточки.

При обновлении Figma-файлов **не добавлять** эти блоки в эталон продакшена, если не принято отдельное продуктовое решение.

---

## 4. Ошибки

- Строка предупреждения `.hero__warn`, если любой из запросов вернул ошибку (показывается первое сообщение из списка).

---

## 5. Состояния для макета

1. **Норма:** есть данные по питанию, финансам, активный квест.
2. **Новичок:** EXP 0/100, нет квестов — показать текст квестов `hub.questText`.
3. **Ошибка сети:** одна строка ошибки под контентом.

---
