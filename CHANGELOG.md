# Changelog

Формат основан на практике [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/). Версии фронтенда совпадают с полем `version` в корневом `package.json`, если не указано иное.

## [0.5.2] — 2026-03-29

### Добавлено

- **Модуль «Задачи»:** API `GET/POST/PATCH/DELETE /api/v1/tasks` (модель `UserTask` в Prisma: заголовок, заметка, локальный срок `dueDate` / `dueTime`, выполнение, задел `source` MANUAL \| QUEST для квестов). Экран с вкладками **Сегодня** (день + просрочка + сделано за день), **Входящие** (без даты), **Все**; форма добавления; бейдж «Квест» для будущих квестовых задач.

## [0.5.1] — 2026-03-29

### Добавлено

- **Экран «Герой»:** уровень и полоса EXP (формула уровней и сохранение `totalExp` в `localStorage` как задел до API), титул по уровню, блок «Сегодня» с переходом на «Тело» и «Задачи».
- **Сводки:** кратко питание за сегодня (КБЖУ к цели) и **всего по финансам** (счета + инвестиции), серии дней подряд по **завершённым тренировкам** и **замерам** (от сегодняшней даты назад).
- **Заготовки:** карточка квеста (пустое состояние), ряд «Скоро» (инвентарь, питомец, мини-игра) — неактивные кнопки.
- **Оболочка:** `ShellTabContext` с `goToTab(index)` и `useShellGoToTab()` для перехода между вкладками с экрана «Герой».

## [0.5.0] — 2026-03-28

### Изменено

- **Главная «Финансы»:** блок «На что тянет пассивный доход» перенесён **между** плитками отчёта (доходы / расходы / баланс) и сводкой «На счетах»; день / неделя / месяц показываются **каруселью** (свайп, стрелки, точки), а не тремя колонками подряд.
- **Инвестиции:** подсказки по пассивному доходу по-прежнему **три блока друг под другом** (без карусели).
- **Тексты примеров (`financeMotivationCards.ts`):** для каждой ступени дневного эквивалента (до **5000 ₽/день**) заданы **конкретные покупки** на сумму дня, недели (~×7) и месяца (~×30); три варианта на ступень, случайный набор при обновлении и смене вкладки.

### Документация

- Обновлены **`ПАСПОРТ_ПРОЕКТА.md`** и корневой **`README.md`** (версия 0.5.0, описание UI пассива).

## [0.4.0] — 2026-03-28

### Добавлено

- **Модуль «Тело»** (`/api/v1/body`, экран с тремя подвкладками):
  - **Замеры:** вес, рост, % жира, обхваты (в БД кг/см); список записей.
  - **Питание:** дневник КБЖУ по приёмам пищи, итоги дня, прогресс к цели по калориям.
  - **Тренировки:** программы с циклом дней, упражнения (сила, кардио, растяжка, разминка, другое), журнал тренировок и строки результата (`resultJson`).
- **Настройки тела:** единицы массы (кг/lb) и длины (см/дюймы); цели на день — калории и БЖУ (г).
- **Мотивационные карточки (финансы):** на главной и во вкладке «Инвестиции» — подсказки «на что тянет пассивный доход» по дню / неделе / месяцу (ступени 5 / 25 / 50 ₽).
- **Финансы — настройки карточек счёта:** что показывать из оценки дохода/расхода по %% (день, неделя, месяц, год); поле `interestIncomeWeekMinor` в API.
- **Бюджет** (вкладка «Финансы»): лимиты по категориям за календарный месяц, сводка на главном экране (при наличии лимитов).

### Изменено

- **Модалки финансов:** полноэкранный режим для удобства на мобильных (`finance__modal--fullscreen`).
- **Категории:** скрыта системная «Универсальная» из UI; подпись «Нет категории» вместо «Без категории»; расширены пресеты эмодзи и цветов; кнопка «В архив» вместо символа «извлечь».
- **Инвестиции:** карусель вкладов на всю ширину; в обзоре и метриках не учитываются кредитные карты и банковские счета с отрицательным балансом (долг).
- **Карусель счетов:** скрытие полосы прокрутки у полноэкранных слайдов; отступы контента от кнопок навигации.
- **API:** `apiFetch` экспортируется из `financeApi.ts` для общих запросов (в т.ч. `bodyApi`).

### Документация

- Обновлены **`ПАСПОРТ_ПРОЕКТА.md`**, корневой **`README.md`**, **`api/README.md`**: модуль «Тело», префикс `/api/v1/body`, бюджет и расширенные настройки финансов в API.

### Заметки для разработки

- После изменений схемы Prisma: `npx prisma db push` и `npx prisma generate` в каталоге `api/`. На Windows в синхронизируемых папках `prisma generate` иногда завершается **EPERM** — закрыть процессы, использующие query engine, или выполнить генерацию вне блокировки.

## [0.3.0] — 2026-03-28

### Добавлено

- **Отчётный период:** день месяца (1–28) в настройках пользователя; сводка доходов/расходов за текущий период на главном экране; API `GET/PATCH .../settings`, `GET .../summary/reporting`.
- **Аналитика:** первая метрика — ожидаемый баланс к последнему дню отчётного периода (`GET .../analytics/reporting-forecast`).
- **Удаление счёта с историей:** `POST .../accounts/:id/merge-into` — перенос операций и переводов на выбранный счёт, затем удаление; модальное подтверждение на главном экране.
- **Финансы → Инвестиции:** портфель с оценкой стоимости, обновление цен по сохранённым котировкам при открытии вкладки (`overview?refresh=1`).
- Структура портфеля: доли и суммы по **вкладам/наличным**, **картам**, **акциям**, **облигациям**, **прочим инструментам** и всего богатства (счета + инвестиции).
- Поиск бумаг: **MOEX** (акции, облигации, фонды РФ) и **CoinGecko** (крипто); подстановка типа актива, цены, при возможности — **купона** (облигации) и **дивидендов за ~12 мес.** (акции) с MOEX.
- Учёт дохода **с одной бумаги в год** (модалка «Доход», метрики портфеля считают итого как × количество); сохранена совместимость со старым полем «доход на всю позицию».
- Привязка позиции к источнику котировки (`quoteSource`, `quoteExternalId`, `quoteMoexMarket`) для повторного запроса цены.

### Изменено

- **Главный экран финансов:** блок итогов капитала, оценка пассивного дохода ~в месяц, настройка отчётного числа и плитки периода; после сделок на вкладке «Инвестиции» данные обновляются без перезагрузки страницы.
- **Вкладка «Инвестиции»:** убрана строка «Капитал всего»; полная разбивка и сумма «всего» остаются на главном экране.
- API: расширены `POST/PATCH` позиций, ответы с `annualCashflowTotalMinor`; эндпоинт `GET .../investments/quote-fundamentals`.
- Документация: заполнены корневой `README`, `ПАСПОРТ_ПРОЕКТА.md`, расширен `api/README.md` (финансовые маршруты).

### Заметки для разработки

- После изменений схемы Prisma: `npx prisma db push` и `npx prisma generate` в каталоге `api/`. На Windows в синхронизируемых папках `prisma generate` иногда завершается **EPERM** — закрыть процессы, использующие query engine, или выполнить генерацию вне блокировки.
