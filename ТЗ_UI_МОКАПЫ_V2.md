# ТЗ: интерфейс по согласованным мокапам (v2)

Документ фиксирует целевое состояние UI **Real Hero** по согласованным изображениям `docs/screenshots/v2/` и текущему функционалу API. Реализация — поэтапно в коде; отклонения только по согласованию.

## 1. Общие принципы

- Тёмная тема: фон `#0f1115` / карточки `#14171c`–`#1a1d24`, акцент синий `#8ab4ff`, доход зелёный, расход красный, тело — зелёный акцент. В коде задано через **CSS-переменные** `:root` в `index.css` (`--rh-bg`, `--rh-surface`, `--rh-accent`, …) — дальнейшую сверку с PNG мокапов делать по этим токенам.
- **Ширина колонки приложения:** `max-width: 428px` (`--rh-shell-max`) — логическая ширина **iPhone 13 Pro Max**, контент по центру; `viewport-fit=cover` и отступы `safe-area` для выреза. Для пиксельной сверки положите эталонные PNG в `docs/screenshots/v2/` и сравнивайте с устройством или эмулятором той же ширины.
- Нижняя навигация с иконками и читаемыми подписями.
- Данные на экранах — из существующих эндпоинтов; без выдуманных полей в API.

## 2. Экран «Герой» (`HubScreen`)

**Согласовано с мокапом v2-01.**

| Элемент | Требование |
|--------|------------|
| Блок уровня / EXP | Без изменений по смыслу: аватар, уровень, титул, полоса EXP, подпись. |
| Карточка «Сегодня» + стрики | **Не показывать** (уже убрано). |
| Мини-карточка «Тело» | Строка 1: ккал за сегодня / цель или только ккал (`fetchNutritionDay` + `fetchBodySettings`). Строка 2: **БЖУ** в виде «Б … · Ж … · У …» г из `totals` (округление до целых). |
| Мини-карточка «Финансы» | Строка 1: всего по счетам + инвестициям (`fetchAccounts`). Строка 2: **оценка пассивного потока ~в месяц** — как на главной финансов: `fetchInvestOverview`: `depositSavingsIncomeMonthMinor + couponDividendMonthMinor`; показывать только если сумма > 0. |
| Квесты | Список активных с кнопками «К задачам», «Отказаться»; доступные с «Взять» / `questStart`; ветка + название. Без лишних блоков из мокапов-не-кода. |

## 3. Финансы → Главная (`FinanceMainPanel`)

**Согласовано с v2-02 и текущей логикой.**

- Карусель счетов: на карточке — баланс, ставка при необходимости, **строки доходности** по настройкам карточки (`accountCardYieldLine`, флаги день/неделя/месяц/год).
- Плитки отчёта за период, мотивация при пассиве > 0, последние операции.
- Блок **полной сводки**: на счетах, инвестиции (оценка), всего, пассив ~в месяц, при наличии — полоска бюджета, **разбивка капитала**.
- Отдельный блок «перейти в инвестиции» не добавлять.

## 4. Финансы → Инвестиции (`FinanceInvestPanel`)

**Согласовано с v2-03.**

- Оценка бумаг, карусель вкладов/накопительных с доходностью.
- Структура портфеля с долями.
- Список метрик: **на 1000 ₽ в год**, %% вкладов в месяц, бумаги день / месяц / год — данные из `fetchInvestOverview().metrics`.

## 5. Финансы → Аналитика и бюджет

- Аналитика: прогноз + разбивка по категориям (как в коде).
- Бюджет: таблица лимитов за месяц (как в коде); визуально читаемые строки.

## 6. Категории расходов/доходов (`CategoryTreePanel`)

**Согласовано с упрощением v2-06.**

- Вкладки Расход / Доход, поиск — сохранить.
- Дерево: строка категории **крупнее**, без «стены» из мелких иконок-кнопок.
- Действия: текстовые ссылки **«Подкатегория»** и **«В архив»** (или эквивалент), не две загадочные кнопки «+» и «⏏».
- Новая категория: поле названия + **«Создать»**; выбор эмодзи и цвета — в **раскрывающемся блоке** «Значок и цвет» (по умолчанию авто из названия и индекса).
- Подкатегория: форма **только название** + создать/отмена; цвет/эмодзи от родителя / авто.

## 7. Добавление операции

- Оставить текущий поток (`AddTransactionPrototype`): тип, сумма, счёт, категория, дата — без лишней плотности.

## 8. Тело → Главная (`BodyHomePanel`)

**Согласовано с v2-08.**

- Вес, график, талия / тренировки за 7 дней — как в коде.
- Добавить явный блок **БЖУ за сегодня** (граммы; при заданных целях в настройках — сравнение с целью или подпись целей).
- Кнопка к замерам: не доминирует — стиль **вторичной ссылки** («История замеров» или аналог).

## 9. Тело → Замеры (`MeasurementsPanel`)

**Согласовано с v2-09.**

- Обязательные к вводу первой очереди: **дата**, **вес**.
- Рост, % жира, талия, заметка — в блоке **«Дополнительные поля»** (`<details>`), по умолчанию свёрнуто.

## 10. Питание, тренировки, задачи

- Питание и задачи — без смены логики; только выравнивание отступов при необходимости после общих токенов.

## 11. Критерии приёмки

- [x] Герой: две строки на карточках Тело/Финансы при наличии данных; пассив совпадает по смыслу с главной финансов.
- [x] Категории: меньше визуального шума, раскрываемые стили, понятные подписи действий.
- [x] Замеры: короткая форма «сверху», детали по желанию в `<details>`.
- [x] Тело главная: блок БЖУ виден при данных за день.
- [x] Нижняя навигация приложения и поднавигации модулей с иконками и подписями; финансы: главная / инвестиции / аналитика / бюджет на ключах `fin.*`; модалка категорий и дерево категорий — i18n.
